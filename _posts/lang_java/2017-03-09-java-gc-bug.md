最近遇到一个频繁gc的服务，记录一下查找问题的过程

## 1. 先用jps 和 jstack定位pid

推荐用如下命令代：

	ps aux|grep java


	[running]mqq@10.242.50.116:~$ jps

	19097 Main
	17075 Main
	20069 Jps
	28296 Main
	18110 Main
	10540 Main

然后根据线程名字（自己找特征）定位查找：

	[running]mqq@10.242.50.116:~$ jstack 28296 |grep 'UserBase'

## 4. 使用jmap -histo[:live] pid查看堆内存中的对象数目、大小统计直方图

如果带上live则只统计活对象

jmap -histo:live 28296 | less
